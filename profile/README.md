# Laraworld
**High-quality Laravel packages with a narrow focus, strong guarantees, and clean developer experience.**

Laraworld is a GitHub organization for Laravel packages that do one thing well.
Every package is designed to be small, practical, well-tested, and easy to understand. The goal is to provide reliable building blocks for Laravel applications without unnecessary abstractions, bloated feature sets, or hidden complexity.

## What to expect
Packages published under Laraworld follow a shared standard:
- **Narrowly scoped**  
  Each package solves a specific problem and avoids becoming a framework within the framework.
  
- **Laravel-first**  
  Built for Laravel applications, conventions, service providers, configuration, testing helpers, and modern Laravel workflows.
  
- **Strictly typed**  
  Compatible with the highest level of PHPStan.
  
- **Fully tested**  
  100% test coverage is expected across the codebase.
  
- **Well documented**  
  Clear installation steps, usage examples, configuration options, and edge cases are documented.
  
- **Actively maintained**  
  Dependencies are reviewed and updated weekly to stay aligned with the Laravel and PHP ecosystems.
  
## Philosophy
Laravel packages should be pleasant to install, simple to use, and safe to rely on.
Laraworld packages are built around a few principles:

1. **Small surface area**
2. **Predictable behavior**
3. **Clear documentation**
4. **Strong static analysis**
5. **Comprehensive tests**
6. **Ongoing maintenance**


The aim is not to publish the most packages possible. The aim is to publish packages that developers can trust.

## For Laravel developers
Use Laraworld packages when you want focused tools that feel native to Laravel and are maintained with care.
Each repository includes its own documentation, examples, tests, and contribution guide.
## Quality baseline
Every Laraworld package aims to support:
- modern PHP versions
- modern Laravel versions
- PHPStan at the highest level
- PHPUnit or Pest test suites
- automated dependency updates
- continuous integration
- clear semantic versioning
- 100% test coverage
## Contributing
Contributions are welcome when they align with the package’s scope.
Before opening a pull request, please read the contribution guide in the relevant repository. Since Laraworld packages are intentionally narrow in scope, not every feature request will be accepted.
Bug reports, documentation improvements, test improvements, and compatibility fixes are always appreciated.
